# 装备系统

> 文件：`scripts/prefabs/mushasword*.lua`, `armor_musha*.lua`, `hat_m*.lua`, `phoenixspear.lua`, `frosthammer.lua`, `bowm.lua`, `pirateback.lua`

## 系统特点

- 所有 Musha 专属装备拥有**独立等级系统**，通过使用积累经验
- 等级通过 `OnSave`/`OnLoad` 持久化
- 装备满级显示 "MAX"
- 燃料耗尽时进入"破损"状态（伤害降至 1）
- 通过 `characterspecific_musha` 组件限制仅 Musha 可使用

| 等级范围 | 护甲值 | 吸收率 |

### 破损冰锤 (Broken Frost Hammer)

| 属性 | 值 |
|------|------|
| 预制件 | `broken_frosthammer` |
| 说明 | 冰锤护甲形态（蝶翼盾） |
| 防雨 | 80% |

冰霜护甲同时承担右键防御和小型冰箱职责，使用独立的 8 格 `frostsmall` 小冰箱容器，不再复用普通 `backpack` 容器逻辑；容器 UI 挂在 `side_inv_behind` 层，额外背包栏里的背包会显示在它上方。冰箱效果通过装备自身的 `preserver` 组件提供，只影响冰霜护甲容器内物品，不给玩家主物品栏附加冰箱效果。

冰霜护甲 prefab 入口仍在 `scripts/prefabs/broken_frosthammer.lua`。第三阶段文件整理已把容器打开/关闭音效、格子常开入口、格挡音效、右键防御开启/关闭、护盾燃料消耗和理智消耗等辅助逻辑抽到 `scripts/musha/prefabs/frost_armor.lua`；等级成长、耐久结算、冰冻反击和保存字段仍由 prefab 入口编排。

额外装备栏兼容由 `scripts/musha/equipment/inventory_overflow.lua` 单独处理。开启额外背包栏时，普通背包优先作为溢出容器：拾取、制作产物和主物品栏 `shift-click` 会先进入额外背包；额外背包已满后，才落入冰霜护甲的小冰箱。未开启额外装备栏、或没有穿戴额外背包时，冰霜护甲仍保持自身小冰箱容器可用。该逻辑只识别 `broken_frosthammer` 和 `BACK`/`PACK`/`BACKPACK`/`BAG` 等额外背包槽，且打开其它箱子、锅、建筑容器等普通容器时不会抢占原版整理目标。

### 海盗背包 (Pirate Back)

| 属性 | 值 |
|------|------|
| 文件 | `pirateback.lua` |
| 预制件 | `pirateback` |
| 类型 | 容器型可穿戴装备 |
| 装备栏 | 默认自动检测额外背包栏；无 `BACK`/`PACK`/`BACKPACK`/`BAG` 时回退身体栏 |

额外背包栏模式下，海盗箱子只占用 `backpack`/`swap_body_tall` 背部显示层，不再覆盖护甲使用的 `swap_body` 身体显示层；强制 Body 模式时保留旧版护甲式外观。
