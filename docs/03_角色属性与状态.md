## 四态切换系统（Bloom System）

基于 Wormwood 的 bloomness 组件实现四种形态：

```
满足态 (Full)     ← hunger >= 160
  ↓ hunger < 160
普通态 (Normal)   ← 默认
  ↓ 激活女武神
女武神态 (Valkyrie) ← 主动技能触发
  ↓ 满月/主动
狂暴态 (Berserk)  ← 满月自动触发或主动
```

### 形态 → 动画 Build 对应表

| 形态 | 默认 Build | K 变体 | SW2 变体 | Old 变体 |
|------|-----------|--------|---------|---------|
| 满足 | `musha` | `musha_full_k` | `musha_full_sw2` | `musha_p_old` |
| 普通 | `musha_normal` | `musha_normal_k` | `musha_normal_sw2` | `musha_normal_p_old` |
| 女武神 | `musha_battle` | `musha_battle_k` | `musha_battle_sw2` | `musha_battle_p_old` |
| 狂暴 | `musha_hunger` | `musha_hunger_k` | `musha_hunger_sw2` | `musha_hunger_p_old` |

外观通过 `scripts/musha_visual_commands.lua` 中的 `visual_hold` RPC 循环切换（12+ 组合状态）。

## 网络变量（Net Variables）

用于服务端→客户端同步的 NetVar：

```lua
inst._fatigue_sleepmax          = net_float  -- 疲劳上限
inst._currentfatigue_sleepperc  = net_float  -- 当前疲劳百分比
inst._stamina_sleepmax          = net_float  -- 耐力上限
inst._currentstamina_sleepperc  = net_float  -- 当前耐力百分比
inst._spellpowermax             = net_float  -- 法力上限
inst._currentspellpowerperc     = net_float  -- 当前法力百分比
```

同步事件：
- `"clientsetfatigue_sleepmax"` / `"clientfatigue_sleeppercentchange"`
- `"clientsetstamina_sleepmax"` / `"clientstamina_sleeppercentchange"`
- `"clientsetspellpowermax"` / `"clientspellpowerpercentchange"`

## 存档系统 (OnSave / OnLoad)

### 保存字段
```lua
data.level            -- 等级
data.levelexp         -- 经验值
data.music            -- 音乐充能
data.count_w          -- 守护者计数
data.treasure         -- 宝藏计数
data.berserk          -- 狂暴标记
data.berserks         -- 狂暴子状态
data.fberserk         -- 完全狂暴
data.active_valkyrie  -- 女武神标记
data.valkyrie (1-14)  -- 女武神14等级
data.berserk_a (1-3)  -- 狂暴护甲3等级
data.shield_aura      -- 护盾光环
data.on_sparkshield   -- 电弧护盾
data.no_bodyguard     -- 保镖禁用
data.dsniff_*         -- 难度设定
-- 所有组件状态 (spellpower, stamina_sleep, fatigue_sleep)
```

OnLoad 恢复所有标记、组件状态、事件监听器、周期任务。
幽灵/人类状态切换监听也通过 `musha_taskutils.lua` 注册，避免多次 OnLoad 后重复绑定 `ms_respawnedfromghost`、`ms_becameghost`、`hungerdelta`、`newstate`。
