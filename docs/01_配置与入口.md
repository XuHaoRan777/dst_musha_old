# 配置项与入口系统

## modinfo.lua 维护方式

`modinfo.lua` 是 DST 运行时读取的单文件入口。为了保持游戏加载兼容性，发布态仍保留根目录 `modinfo.lua`。

配置源码已拆到 `modinfo_src/`：

| 路径 | 内容 |
|------|------|
| `modinfo_src/meta.lua` | Mod 基础元信息 |
| `modinfo_src/options/language.lua` | 语言配置 |
| `modinfo_src/options/keys.lua` | 快捷键配置 |
| `modinfo_src/options/difficulty.lua` | 难度配置 |
| `modinfo_src/options/recipes.lua` | 配方相关配置 |
| `modinfo_src/options/character.lua` | 角色机制配置 |
| `modinfo_src/options/visual.lua` | 外观配置 |
| `modinfo_src/options/containers.lua` | 容器配置 |
| `modinfo_src/options/misc.lua` | 其他杂项和历史注释配置 |

生成命令：

```powershell
.\tools\build_modinfo.ps1
```

检查命令：

```powershell
.\tools\build_modinfo.ps1 -Check
```

生成和检查命令都会执行 `luac -p`，覆盖 `modinfo_src/meta.lua`、全部 `modinfo_src/options/*.lua` 和生成后的 `modinfo.lua`。因此配置源文件必须保持可被 Lua 语法检查单独解析。

不要直接手改 `modinfo.lua` 的配置项；应修改 `modinfo_src/` 后重新生成。

## modinfo.lua 配置项总览（31+ 项）

### 语言设置

| 配置名 | 标签 | 默认值 | 选项 |
|--------|------|--------|------|
| `modlanguage` | In-game Language | `english` | english, korean, chinese, russian |

该配置只影响游戏内文本，配置页面自身仍根据客户端语言显示中文或英文。旧配置残留的 `auto` 会按 `english` 兜底，不再扫描语言包自动切换。

## modmain.lua 入口结构

### 加载顺序

```
1. 读取全局引用 (GLOBAL.require, GLOBAL.STRINGS 等)
2. `require("musha_config").Load(GetModConfigData)` 读取主要配置，并写入 `TUNING.MUSHA.KEY*` 与 `TUNING.MUSHA_PIRATEBACK_SLOT`
3. modimport("libs/env.lua")         ← 模块加载器
4. modimport("scripts/prefabasset.lua")   ← 资源声明
5. modimport("scripts/musha_adds_recipe.lua") ← 配方
6. 语言检测 + 加载语言文件
7. modimport("scripts/musha_adds_states.lua") ← 状态图
8. modimport("scripts/musha_adds_actions.lua") ← 行为
9. modimport("scripts/musha_adds_container.lua") ← 容器
10. `musha_inventory_overflow` 注册冰霜护甲/额外背包栏的溢出容器优先级 Hook
11. `musha_config_postinit` 注册角色配置、材料燃料、口味等通用 PostInit
12. modimport("scripts/mypower_musha_1.lua")    ← 法力状态
13. modimport("scripts/widgets/...")              ← UI 注入
14. modimport("scripts/difficulty_monster_dst.lua") ← 全服怪物难度
15. use "libs/mod_env"(env)                       ← 环境设置
16. use "data/widgets/controls"                   ← 加载控件 Hook
17. use "data/screens/chatinputscreen"            ← 加载聊天输入 Hook
18. use "data/screens/consolescreen"              ← 加载控制台 Hook
19. `musha_equipment_postinit` 注册装备外观、冰霜护甲、Dall 生成和冰剑配置 PostInit
20. `musha_info_commands` 注册等级/技能信息显示 RPC
21. `musha_skill_lightning` 注册闪电 RPC 与 R 键入口相关的主动闪电逻辑
22. `musha_skill_shadow` 注册暗影/隐身 RPC、潜行攻击和被发现清理逻辑
23. `musha_skill_sleep` 注册睡眠 RPC 与深睡/浅睡/唤醒逻辑
24. `musha_skill_shield` 注册主动护盾 RPC 与冷却/范围伤害逻辑
25. `musha_skill_music` 注册音乐/嗅探 RPC、宝藏生成和音乐召唤逻辑
26. `musha_companioncommands` 注册 Yamche、Arong、Dall 命令/RPC
27. `musha_world_postinit` 注册世界物件 tag、命中特效、植物 follower 等 PostInit
28. `musha_visual_commands` 注册外观切换 RPC
29. `musha_companion_hoverinfo` 注册 Yamche/Arong 血量悬浮信息 UI Hook
30. `musha_difficulty_postinit` 注册角色难度、死亡惩罚、闪电特效和禁食物 PostInit
```

### 原版预制件修改（AddPrefabPostInit）

PostInit 注册已从 `modmain.lua` 拆到专门模块；`modmain.lua` 只负责加载这些模块：

| 模块 | 修改内容 |
|-----------|---------|
| `scripts/musha_config_postinit.lua` | `musha` bodyguard/diet/taste/wildness 配置；材料 fuel 组件与 `elements` tag；`tentacle_pillar_arm` 不给经验；`icecream` tag |
| `scripts/musha_equipment_postinit.lua` | `armor_mushaa`、`armor_mushab`、`pirateback` 外观配置；`broken_frosthammer` 蝶翼护盾配置；`moontree_musha` 生成配置；`mushasword_frost` 三段强化配置 |
| `scripts/musha_inventory_overflow.lua` | 冰霜护甲 `frostsmall` 小冰箱与额外背包栏的溢出顺序：背包优先，背包满后落入冰霜护甲；同时处理拾取、制作产物和主物品栏 `shift-click` |
| `scripts/musha_world_postinit.lua` | `hound`/`frog` 命中特效符号；`nightstick` 电武器 tag；`slurtlehole` no_target；`tentacle_pillar_arm` arm；`farm_plant_randomseed` 和 `green_mushroom` follower/tag |
| `scripts/musha_companion_hoverinfo.lua` | Yamche/Arong 的 `healthinfo_copy`、`hungerinfo`、`health_replica` 与 hover 文本注入 |
| `scripts/musha_difficulty_postinit.lua` | `musha` 角色难度 flag、近战倍率、远程武器 tag、死亡惩罚、睡眠 UI 选项、闪电特效档位、禁食物 tag |

### 信息显示模块

`scripts/musha_info_commands.lua` 负责 `INFO`、`INFO2` RPC。`INFO` 显示等级、经验和睡眠/疲劳/音乐/嗅探资源百分比，并标记跟随中的 Yamche/Critter 展示信息；`INFO2` 显示技能解锁进度。该模块只影响文字反馈，不修改角色数值或保存字段。

### 伴侣命令模块

`scripts/musha_companioncommands.lua` 负责注册以下 RPC：`dall`、`arong`、`yamche`、`yamche2`、`yamche3`。

该模块复用 `scripts/musha_companionstates.lua` 的查找、跟随、休息和跨世界迁移封装；`modmain.lua` 只传入 `AddModRPCHandler`、`STRINGS`、`TheSim`、`SpawnPrefab`、`MushaCommands` 等运行环境对象，不再直接保存大段伴侣命令逻辑。

### 主动护盾模块

`scripts/musha_skill_shield.lua` 负责 `on_shield_act` RPC，以及主动护盾的充能、激活、范围伤害、冷却和回收逻辑。
它保留了原有 `activec0`、`on_sparkshield`、`shield.ready_flag` 等状态字段，便于和旧存档与现有战斗逻辑兼容。

### 闪电技能模块

`scripts/musha_skill_lightning.lua` 负责 `Lightning_a` RPC，以及 R 键入口关联的主动闪电、命中回调、档位伤害、幽灵复仇闪电、魔力消耗/返还、Yamche 闪电状态同步和武器电击 stimuli 维护。
该模块仍保留 `switch`、`active_valkyrie`、`lightning_spell_cost`、`vl1-vl8` 等旧状态字段，避免破坏已有状态判断。

### 睡眠技能模块

`scripts/musha_skill_sleep.lua` 负责 `sleeping` RPC、深度睡眠、浅睡眠、唤醒、睡眠前战斗状态清理和输入状态保护逻辑。
该模块仍保留 `sleep_on`、`tiny_sleep`、`musha_press`、`music_check`、`switlight` 等旧状态字段，避免影响资源回复周期任务和音乐技能入口。
### 暗影技能模块

`scripts/musha_skill_shadow.lua` 负责 `shadows` RPC、隐身启动/取消、潜行攻击结算、被攻击发现、敌人目标清理和失败反馈。
该模块仍保留 `sneak_pang`、`sneaka`、`poison_sneaka`、`in_shadow`、`notarget` 等旧状态字段；睡眠模块会通过 `ShadowSkill.ClearState()` 统一清理隐身事件监听，避免隐身回调残留。
### 音乐/嗅探技能模块

`scripts/musha_skill_music.lua` 负责 `buff` RPC、X 键嗅探、宝藏/蛋生成、三种音乐演奏、临时光环、幽灵犬/暗影分身/冰触手召唤和 Wormwood 植物特效任务。
该模块仍保留 `music`、`treasure`、`treasure_sniffs`、`switlight`、`start_music`、`music_armor`、`lightaura`、`moondrake_on`、`planttask` 等旧状态字段，避免影响保存读取、资源显示和现有周期任务。
### 外观切换模块

`scripts/musha_visual_commands.lua` 负责 `visual_cos`、`visual_human`、`visual_hold` RPC。外观循环已经从 `modmain.lua` 的长 `elseif` 分支收敛为固定顺序表，但仍保留 `visual_hold*`、`full_hold`、`normal_hold`、`valkyrie_hold`、`berserk_hold`、`hold_old*`、`*_k_hold`、`willow`、`wigfred`、`change_visual` 等旧字段，避免影响已有存档和角色主体中的外观判断。
