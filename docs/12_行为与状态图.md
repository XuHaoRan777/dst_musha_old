## 一、自定义 Action 覆盖

Musha mod 不创建全新 Action，而是**覆盖原版 Action 的 fn 函数**，在原有逻辑中插入 Musha 特有分支。

### 覆盖的 Action 列表

#### USEITEM

**原始优先级不变，重写 fn 函数：**

| 分支 | 条件 | 行为 |
|------|------|------|
| Musha 装备切换 | `musha_items` tag + `machine` 组件 + 非 crown | `TurnOn/TurnOff` 切换 boost 状态 |
| Musha Crown | `musha_items` tag + `mushacrown` | `useableitem:StartUsingItem()` |
| 原版 | 非 `musha_items` | 原版 `useableitem` 行为 |

#### TURNON

**优先级改为 3，重写 fn + strfn：**

| 分支 | 条件 | 行为 |
|------|------|------|
| Musha 物品 | `musha_items` tag | 直接返回 false（阻止） |
| 普通机器 | 非 critter | 原版 `machine:TurnOn` |
| Critter（宠物）+ despawn 模式 | `act.doer.despawn = true` | 两步确认后释放宠物 |
| Critter + Woby | `tar.woby` | 触发 Woby 变身 + 60s 冷却 |
| Critter + 工作模式 | 非 Woby + owner match | `machine:TurnOn`（启动收集） |
| Critter + 背包满 | container:IsFull() | 宠物说"满了" |

**strfn：** critter 时显示 `CRITTER_ACTION_TURNON`，否则显示原版文字。

#### ADDFUEL

**重写 strfn：** `musha_items` tag 时文字变为 `MUSHA_ACTION_REPAIR`（修复），否则原版"添加燃料"。

**fn 逻辑不变：** 移除燃料物品 → `fueled:TakeFuelItem`。

#### RUMMAGE

**完全重写 fn，支持两种容器系统：**

| 分支 | 条件 | 行为 |
|------|------|------|
| 普通容器 | 非 critter + 非 yamcheb | 原版开/关容器逻辑 |
| 宠物/Yamche 容器 | critter/yamcheb tag | 仅 leader 或 house 标记者可开 |

#### PICKUP

**完全重写 fn，区分拾取者：**

| 拾取者 | 目标容器 | 行为 |
|--------|---------|------|
| critter_musha / yamcheb | `components.container` | 放入自身容器 |
| 普通玩家 | `components.inventory` | 原版拾取逻辑（含 YOTC、重物、蜘蛛检查） |

#### HARVEST

**完全重写 fn，区分收割者：**

| 收割者 | 行为 |
|--------|------|
| critter_musha / yamcheb | 收割后放入自身 container |
| 普通玩家 | 收割后放入 inventory |

#### FEED

**完全重写 fn，区分喂食目标：**

| 目标 | 条件 | 行为 |
|------|------|------|
| 非 critter_musha | trader 组件 | `trader:AcceptGift` |
| 非 critter_musha | eater 组件 | `eater:Eat` + 投毒检测 |
| critter_musha + Woby | — | 直接喂食 |
| critter_musha（非 Woby） | perishable ≤ 95% | 喂食 + `master_gift = true` |
| critter_musha（非 Woby） | perishable > 95% | 拒绝 + 随机拒绝台词 |

### Action 优先级调整

```lua

```

#### book22 状态时间线

```
0 FRAMES    → 开始
28 FRAMES   → 播放翻书音效
54 FRAMES   → 合书音效 + 推送 "musha_book_done" 事件
58 FRAMES   → 退出状态
```

#### Lightning 状态时间线

```
进入        → 添加暗化覆盖层 (ColourAdder)
攻击帧      → 触发 combat:DoAttack
冷却        → max(combat_min_period + 0.5 FRAMES, 6 FRAMES)
退出        → 移除覆盖层
```

## 五、代码质量问题

### 重复代码

`musha_adds_actions.lua` 中 PICKUP / HARVEST / RUMMAGE 的 critter 与普通玩家分支存在大量重复逻辑，仅区别在于 `container` vs `inventory` 目标。重构时应提取公共逻辑。

### 全局函数泄漏

Action 覆盖中的 `CanEntitySeeTarget` 直接引用全局，未通过 `GLOBAL.` 前缀。

### 原版覆盖风险

直接重写 `ACTIONS.*.fn` 会覆盖**所有角色**的 Action 行为（即使非 Musha 玩家也受影响），重构时应改为条件分支或 PostInit 补丁。
