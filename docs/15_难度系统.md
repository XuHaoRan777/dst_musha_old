# 难度系统

> 文件：`scripts/difficulty_monster_dst.lua`

## 概述

通过 `difficultover` 配置项选择怪物强化档位。系统使用表驱动的 `AddPrefabPostInit` 注册，在怪物生成后修改其 health / combat / locomotor 属性，并添加分级标签供其他系统识别。

**配置选项：** `monster2x` / `monster3x` / `monster4x`。无选择时不改数值，但会给 Boss/精英添加 `giant1x` / `small_giant1x` 标签，保证经验奖励分支可达。

## 标签系统

难度系统为怪物添加分级标签，其他系统可据此判断：

| 标签 | 含义 | 用途 |
|------|------|------|
| `giantNx` | 强化 BOSS | 等级经验加成、掉落判断 |
| `small_giantNx` | 强化精英 | 同上 |
| `monsterNx` | 强化普通怪 | 同上 |

其中 `N` = 2/3/4，对应难度档位。

## 实现特点与问题

1. **已改为表驱动注册** — 难度配置集中在 `monster_sets` 表中，通过统一的 `ApplyMonsterDifficulty()` 应用，避免 80+ 个全局函数污染环境。

2. **已修复已知标签/Prefab 错误** —
   - 海象 prefab 使用 `walrus`，不再是错误的 `walus`
   - 4x 眼球草使用 `monster4x`，不再误标为 `monster2x`
   - 2x/3x 眼球草补齐 `monster2x` / `monster3x` 标签

3. **仍需确认的数值不一致** —
   - 2x 的 merm 血量是 ×4 而非 ×2
   - 3x/4x 小怪血量倍率（×4/×6）与档位名称（3x/4x）不符
   - 部分 BOSS 使用 `DEERCLOPS_DAMAGE` 作为基准而非自身伤害常量

4. **所有玩家受影响** — 难度增强通过 `AddPrefabPostInit` 应用，影响服务器上所有玩家（非 Musha 专属），重构时可考虑是否需要隔离。
