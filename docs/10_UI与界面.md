# UI 与界面系统

> 文件：`scripts/widgets/`, `data/screens/`, `data/widgets/controls`, `scripts/*statusdisplays.lua`

## 概述

Musha mod 通过两种方式扩展 UI：
1. **自定义 Widget** — 三条资源的 Badge（法力/疲劳/耐力）
2. **原版文件覆盖** — 直接替换 `data/screens/` 和 `data/widgets/` 下的原版文件（**高风险**，游戏更新易失效）

---

## 自定义资源 Badge

### 法力 Badge (SpellpowerBadge)

| 属性 | 值 |
|------|------|
| 文件 | `scripts/widgets/spellpowerbadge.lua` |
| 注入文件 | `scripts/spellpower_statusdisplays.lua` |
| 父类 | `Badge` |
| 动画 Bank | `status_meter` / `status_health` |
| 颜色 | 紫色 (UICOLOURS.HEALTH) |
| NetVar | `net_float` → `spellpower` |
| 位置 | StatusDisplays 中动态偏移 |

**结构：**
- 圆形仪表盘 + 数值文字
- 通过 `AddClassPostConstruct("widgets/statusdisplays")` 注入
- 监听 `spellpowerdelta` 事件更新显示
- Badge 内显示当前值/最大值

### 疲劳 Badge (FatigueSleepBadge)

| 属性 | 值 |
|------|------|
| 文件 | `scripts/widgets/fatigue_sleepbadge.lua` |
| 注入文件 | `scripts/fatigue_sleep_statusdisplays.lua` |
| 父类 | `Badge` |
| 颜色 | 黄色系 |
| NetVar | `net_float` → `fatigue_sleep` |
| 方向 | 值越高表示越疲劳（与法力/耐力相反） |

### 耐力 Badge (StaminaSleepBadge)

| 属性 | 值 |
|------|------|
| 文件 | `scripts/widgets/stamina_sleepbadge.lua` |
| 注入文件 | `scripts/stamina_sleep_statusdisplays.lua` |
| 父类 | `Badge` |
| 颜色 | 蓝色系 |
| NetVar | `net_float` → `stamina_sleep` |

### Badge 注入机制

```lua
-- 在 modmain.lua 中通过 AddClassPostConstruct 注入
AddClassPostConstruct("widgets/statusdisplays", function(self)
    -- 创建 Badge 实例
    -- 设置位置偏移（相对于原版 HUD 元素）
    -- 绑定 NetVar 监听
end)
```

三个 Badge 的注入文件结构相同：
1. `require` 对应的 Badge widget
2. 通过 `AddClassPostConstruct` 在 StatusDisplays 构造后追加
3. 读取 `inst.replica` 上的 NetVar 同步值
4. 定期或事件驱动刷新显示

## 原版文件覆盖（高风险区域）

### 覆盖的 Screen 文件

| 文件路径 | 原版路径 | 修改说明 |
|---------|---------|---------|
| `data/screens/chatinputscreen.lua` | `screens/chatinputscreen.lua` | 聊天输入界面 — 可能添加快捷命令或表情 |
| `data/screens/consolescreen.lua` | `screens/consolescreen.lua` | 控制台界面 — 可能添加调试命令 |

### 覆盖的 Widget 文件

| 文件路径 | 原版路径 | 修改说明 |
|---------|---------|---------|
| `data/widgets/controls.lua` | `widgets/controls.lua` | 主控制面板 — 可能修改 HUD 布局以容纳新 Badge |

### 风险说明

**这是 Musha mod 最大的兼容性隐患：**

1. 这些文件目前通过自定义 `use` 加载，并在文件内调用 `AddClassPostConstruct` 做增量 Hook。
2. 虽然不是全文替换原版类定义，但仍然占用了同类 Screen/Widget hook，和其他修改同类界面的 mod 有冲突风险。
3. 聊天/控制台界面不再通过 `SetPause(true)` 阻止快捷键，改由 `keyhandler` 判断当前是否有文本输入界面。
4. `keyhandler` 需要识别聊天、控制台、路牌/命名项圈等 `writeable` 文本输入；输入界面退出时必须释放 `TheFrontEnd:LockFocus(false)`，否则 Musha 快捷键会被客户端焦点卡死。
5. 服务端技能/伴侣 RPC 不应按附近 `_writeable` 实体直接吞掉热键；是否正在文本输入由客户端 `keyhandler` 判断。
6. 后续仍建议逐步把 `data/screens/` 和 `data/widgets/` 下的 Hook 迁回 `scripts/widgets/` 或专门的 UI hook 文件，降低“data 覆盖”误解和维护成本。

**重构建议：**
- 将 UI hook 从 `data/` 目录迁移到命名更明确的 `scripts/ui_hooks/`。
- `controls.lua` 当前只挂了一个空按键监听，可继续清理或移除加载。
- 聊天/控制台 hook 应保持只处理输入焦点，不再直接暂停世界。

## 其他 UI 相关

### 伴侣悬浮信息

`scripts/musha/companions/hoverinfo.lua` 负责 Yamche/Arong 的悬浮信息：

- 给 Yamche 添加 `healthinfo_copy` 与 `hungerinfo`，显示生命/饥饿百分比。
- 给 Arong 添加 `healthinfo_copy`，显示生命百分比。
- Hook `components/health_replica`，在生命变化时刷新文本。
- Hook `widgets/hoverer`，鼠标悬停到带 `healthinfo_copy` 的实体时，把百分比追加到悬浮文字。

该模块从 `modmain.lua` 拆出，只负责 UI 文本和组件注入，不改变伴侣 AI、跟随或战斗逻辑。

### 快捷键 UI

通过 `keyhandler` 组件和 `modmain.lua` 中的 `TheInput:AddKeyHandler` 实现：

| 快捷键 | 默认值 | 功能 |
|--------|--------|------|
| R | KEY_R | 闪电/女武神技能 |
| G | KEY_G | 暗影/隐身 |
| C | KEY_C | 护盾 |
| X | KEY_X | 音乐/嗅探 |
| T | KEY_T | 睡眠/唤醒 |
| Z | KEY_Z | Yamche 召唤 |
| V | KEY_V | Yamche 攻击/收集 |
| B | KEY_B | Yamche 箱子 |
| F1 | KEY_F1 | Arong 骑乘 |
| F2 | KEY_F2 | Dall 互动 |
| F3 | KEY_F3 | Musha 变身 |
| F4 | KEY_F4 | 特殊功能 |

所有快捷键可通过 mod 配置界面自定义。

### 技能冷却/状态 UI

技能状态通过角色实例上的标记变量（`inst.skill_ready`, `inst.music` 等）管理，在 HUD 上通过以下方式反馈：
- 文字提示（`inst.components.talker:Say()`）
- 动画特效（角色身上的粒子/光效）
- Badge 数值变化（法力消耗可视）

### 疲劳 Debuff 视觉效果

疲劳系统在不同阈值显示叠加的 debuff 动画：
- 30% — `sleep_debuff_30` 轻微困倦效果
- 50% — `sleep_debuff_50` 明显疲惫效果
- 70% — `sleep_debuff_70` 严重疲劳效果
- 90% — `sleep_debuff_90` 极度疲劳效果（屏幕昏暗）
